# AA98-Bl017 Tool Window Placement Rules Baseline

## Parent
- Feature: `DevOps/Features/AA98-F15-Tool-Window-Placement-Rules.md`
- Epic: `DevOps/Epics/AA98-E04-Docking-and-Layout.md`
- Vision: `DevOps/Vision.md`

## Scope
Define the first tool window placement rules baseline for `AA98_AvlnCodeStudio` so contributed workbench surfaces can be placed predictably and consistently within the docking model.

## Goals
- Define the first default placement rules for tool-style workbench surfaces.
- Clarify default host areas and placement expectations.
- Introduce simple placement hints or metadata concepts.
- Keep the baseline aligned with docking host evolution and internal component contributions.

## Assumptions
- Predictable defaults are more important than flexibility in early increments.
- Placement guidance should remain simple while the docking model is still maturing.
- Rich end-user placement customization belongs to later increments.

## Open Questions
- Which placement concepts must be standardized first?
- How closely should placement rules align with component UI contribution contracts from the beginning?
