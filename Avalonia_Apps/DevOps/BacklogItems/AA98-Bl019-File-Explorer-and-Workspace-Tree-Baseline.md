# AA98-Bl019 File Explorer and Workspace Tree Baseline

## Parent
- Feature: `DevOps/Features/AA98-F18-File-Explorer-and-Workspace-Tree.md`
- Epic: `DevOps/Epics/AA98-E05-Workspace-and-Project-Handling.md`
- Vision: `DevOps/Vision.md`

## Scope
Define and introduce the first file explorer and workspace tree baseline for `AA98_AvlnCodeStudio` so users can browse files and folders within an open workspace through a stable navigation model.

## Goals
- Define the first workspace tree model for files and folders.
- Introduce baseline navigation within an open workspace.
- Connect explorer interactions to editor opening workflows.
- Keep the first baseline clear and practical before richer project semantics are added.

## Assumptions
- A straightforward folder-oriented tree is the best first step.
- Explorer behavior should optimize common editing workflows before advanced metadata scenarios.
- Rich project overlays and source-control-aware tree behavior belong to later increments.

## Open Questions
- How much filtering or grouping is needed in the first workspace tree?
- Should hidden files or generated artifacts be shown by default in the first increment?
