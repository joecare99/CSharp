# AA98-Bl020 Recent Workspaces Baseline

## Parent
- Feature: `DevOps/Features/AA98-F19-Recent-Workspaces.md`
- Epic: `DevOps/Epics/AA98-E05-Workspace-and-Project-Handling.md`
- Vision: `DevOps/Vision.md`

## Scope
Define and introduce the first recent workspaces baseline for `AA98_AvlnCodeStudio` so users can quickly return to previously used workspace contexts across sessions.

## Goals
- Define the first recent workspaces model.
- Record recently opened workspaces in a simple local form.
- Provide a baseline workflow for reopening recent workspaces.
- Keep the feature lightweight, reliable, and aligned with later startup improvements.

## Assumptions
- A local recent list is sufficient for the first increment.
- Recent workspace behavior should remain simple and predictable.
- Advanced ranking, roaming, and metadata enrichment belong to later increments.

## Open Questions
- How many recent entries should be retained in the first version?
- Should recent workspaces be exposed in startup UI, menu surfaces, or both?
