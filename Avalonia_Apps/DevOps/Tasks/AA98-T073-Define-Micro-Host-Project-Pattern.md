# AA98-T073 Define Micro Host Project Pattern

## Parent
- Backlog Item: `../BacklogItems/AA98-Bl048-Component-Micro-Host-Foundation.md`

## Goal
Define the common project and composition pattern for AA98 micro hosts.

## Scope
- Define naming, project layout, startup, DI, and validation conventions.
- Keep host apps thin and component-focused.
- Decide which shared helpers belong in reusable layers.

## Execution Notes
1. Review existing host projects before defining the pattern.
2. Document when console-oriented hosts are acceptable.
3. Keep the pattern Linux-first but not Linux-only.

## Acceptance Criteria
- Micro-host pattern is documented and actionable.
- Future host tasks can follow the same conventions.

## Validation
- Review against `AA98-F39 Component Micro Hosts`.

## Status
- Proposed
