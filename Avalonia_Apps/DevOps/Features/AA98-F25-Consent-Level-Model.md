# AA98-F25 Consent Level Model

## Parent
- Epic: `DevOps/Epics/AA98-E07-Privacy-Security-and-Consent.md`
- Vision: `DevOps/Vision.md`

## Goal
Define the first consent level model so `AA98_AvlnCodeStudio` can express user-controlled privacy choices in a simple, understandable, and extensible way.

## Scope
- Define baseline consent levels for external data sharing.
- Clarify how user approval is expressed for AI-related actions.
- Keep the model understandable for both simple and advanced users.
- Prepare the path for later consent refinements.

## Included
- Consent level baseline
- User approval concepts
- Simple and advanced consent expressions
- Extensibility path for later privacy controls

## Excluded for Now
- Enterprise policy management
- Organization-wide governance
- Full compliance mapping
- Complex multi-party consent flows

## Success Indicators
- Users can understand and select consent levels clearly.
- AI-related features can query consent consistently.
- The model remains simple enough for early use and flexible enough for later refinement.

## Candidate Backlog Items
- Define baseline consent levels
- Clarify user approval expressions for AI actions
- Keep the consent model understandable for simple and advanced users
- Prepare the model for later privacy control refinements

## Assumptions
- Consent should be explicit and user-controlled.
- Early consent options should favor clarity over exhaustive complexity.

## Open Questions
- Which consent levels are needed in the first version?
- How should consent map to future chat and context-sharing features?
