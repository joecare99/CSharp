# AA98-F23 Editor Selection and Context Action Model

## Parent
- Epic: `DevOps/Epics/AA98-E06-LLM-Integration-Architecture.md`
- Vision: `DevOps/Vision.md`

## Goal
Define the first editor selection and context action model so later AI-assisted actions can operate on deliberate user-selected content and document context under clear boundaries.

## Scope
- Define the baseline model for selection-based and context-based AI actions.
- Clarify what editor and workspace context may be exposed to later AI features.
- Keep the first model aligned with explicit user control and consent principles.
- Prepare the path for later actions on selections, files, or workspace context.

## Included
- Selection/context action baseline
- Boundaries for editor and workspace context usage
- Alignment with user-controlled invocation
- Extensibility path for later AI-assisted actions

## Excluded for Now
- Rich automated code transformations
- Implicit background AI actions
- Full provider execution flows
- Broad autonomous context gathering

## Success Indicators
- Later AI actions can build on a clear and explicit context model.
- Context boundaries remain understandable and enforceable.
- The design stays compatible with privacy and consent requirements.

## Candidate Backlog Items
- Define the baseline model for selection and context actions
- Clarify which editor and workspace context may be exposed
- Align the model with explicit user-triggered workflows
- Keep the baseline ready for later AI action components

## Assumptions
- Early AI actions should be explicit and user-triggered.
- Selection- and file-oriented actions are a safer early direction than broad automation.

## Open Questions
- Which context scopes should exist from the first version onward?
- How should context boundaries align with future consent levels?

## Status
- Proposed
