# DevOps Planning

This directory contains local planning artifacts that mirror an Azure DevOps backlog structure.

## Structure

- `Epics` contains large work packages
- `Features` contains slices of epic scope
- `BacklogItems` contains actionable backlog entries
- `Tasks` contains implementation-level work items
- `Projects` contains per-project documentation and status notes
- `Roadmaps` contains milestone and roadmap planning notes

## Placement Guidance

- Put cross-cutting strategy and large program items in `Epics`
- Put scenario-specific slices in `Features`
- Put implementable items in `BacklogItems`
- Put implementation work in `Tasks`
- Put project-level documentation close to the named project in `Projects`
- Put milestone and roadmap documents in `Roadmaps`
