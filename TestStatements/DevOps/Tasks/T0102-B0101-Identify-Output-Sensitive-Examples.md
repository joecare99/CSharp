# T0102 - Identify Output-Sensitive Examples

## Parent Backlog Item

- B0101 - Document Expected Output for Learning-Critical Examples

## Summary

List the examples where output, ordering, or observable behavior is essential to understanding the sample.

## Priority

This task is part of the first active implementation step in the current roadmap.

## Execution Focus

- Identify output-sensitive example groups in the current code base
- Distinguish deterministic output from illustrative or timing-sensitive behavior
- Highlight examples where expected output is essential for comparison or learning

## Done Criteria

- Output-sensitive areas are listed.
- Reasons are given for why output needs explicit documentation.
- Deterministic and illustrative outputs are distinguished.
